# SPEC 검증

명세서 기반 작업 완료 후 검증합니다.

## 워크플로우

### 1단계: 명세서 대비 검증
SPEC.md 파일을 읽고 다음을 검증합니다:
1. 명세서의 모든 요구사항이 구현되었는지 확인
2. 각 항목별 체크:
   - [ ] 기능적 요구사항
   - [ ] 비기능적 요구사항
   - [ ] UI/UX 명세
   - [ ] API 설계 (해당 시)
   - [ ] 보안 고려사항
3. 누락된 항목이 있으면 구체적으로 명시
4. 개선 제안 사항 포함

결과를 SPEC-REVIEW.md 파일로 작성합니다.

### 2단계: 피드백 반영
누락 사항이 있으면:
1. SPEC-REVIEW.md 내용을 사용자에게 보여줌
2. 수정 여부 확인
3. 승인 시 누락 사항 구현

### 3단계: 검증 체크리스트
```markdown
### 기능 완성도
- [ ] 모든 기능적 요구사항 구현됨
- [ ] 모든 엣지 케이스 처리됨

### 코드 품질
- [ ] 타입 안전성 확보
- [ ] 린트 규칙 준수
- [ ] 테스트 커버리지 충분

### 보안
- [ ] 입력 검증
- [ ] 인증/권한 확인
- [ ] 민감 정보 보호
```

## 핵심 원칙
1. **객관적 검증**: 명세서 대비 실제 구현 비교
2. **구체적 피드백**: "안 됨" 대신 "X 기능의 Y 부분 누락"
3. **반복 가능**: 피드백 -> 수정 -> 재검증 사이클
